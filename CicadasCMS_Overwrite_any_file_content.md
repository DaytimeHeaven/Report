## Vulnerability Description

A security vulnerability exists in the template management module of CicadasCMS v1.0. The `/system/cms/template/save` interface fails to adequately filter and escape the user-input `filePath` and `content` parameters when handling template content saving logic. This allows attackers to input any existing file path on the system. The interface then saves the value of `content` into the `filePath` file, overwriting the original content of `filePath`, resulting in an arbitrary file content overwriting vulnerability.
## Vulnerability Analysis

The vulnerability's entry point class file is: src/main/java/com/zhiliao/module/web/cms/TemplateController.java
Line 52, within the save() function, there is no validation; it directly passes the templateFile object into the logic layer on line 53.

## Vulnerability remediation recommendations

Path validity verification (core protection): Before writing to a file, `getCanonicalPath()` must be called to obtain the canonical path, and `startsWith()` must be used to verify that the path is located within the preset template storage directory, completely preventing `../` traversal behavior.

Filename/path parameter decoupling: Directly passing physical paths in interface parameters is prohibited. It is recommended to use a "file ID" or "UUID" mapping mechanism. The backend retrieves the real file path from the database based on the ID, preventing attackers from constructing arbitrary paths.

Forced suffix whitelisting: Restricting the overwriting of only files with specific suffixes (e.g., .html, .tpl) through code, prohibiting operations on sensitive files such as .jsp, .java, and .xml that may lead to privilege escalation or system crashes.

File attribute checking: Before writing, determine whether the target file is a critical system file or a read-only file.

System permission hardening: Adhere to the principle of least privilege, ensuring that system accounts running Java applications do not have write permissions outside the web root directory (e.g., /etc, /root).
